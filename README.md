# Вычитка — сверка перевода

Локальный инструмент для вычитки перевода: показывает рядом арабский оригинал,
ваш редактируемый русский перевод и (по желанию) другие варианты перевода для
сравнения. Работает прямо с файлами на диске — данные остаются у вас.

Это живая справка: будет дополняться по мере развития инструмента.

## Что нужно

Перевод можно держать в одном из двух мест (или в обоих сразу — это один и тот
же репозиторий):

- **Локальная папка** — нужен десктопный **Chrome** или **Edge**
  (File System Access API).
- **Репозиторий на GitHub** — работает в любом браузере, **в том числе на
  телефоне**; каждое сохранение становится коммитом.

## Как запустить

Откройте приложение одним из способов:

- по ссылке на размещённую версию (если она развёрнута на хостинге), либо
- открыв локальный файл `vychitka.html` (двойным кликом, Chrome/Edge).

Дальше подключите перевод:

- **«Открыть папку»** — выберите корневую папку перевода на диске. Приложение
  один раз спросит доступ; дальше читает и пишет файлы прямо на диске. Данные
  никуда не уходят.
- **«Открыть с GitHub»** — укажите репозиторий, ветку (можно оставить пустой)
  и токен доступа (см. раздел ниже). Так инструмент работает откуда угодно,
  включая смартфон.

При следующем запуске появится кнопка «Открыть …» — она переоткрывает то, с чем
вы работали в прошлый раз.

### Перевод на GitHub (онлайн-режим)

Та же папка перевода, только лежит в репозитории GitHub (удобно сделать его
приватным). Структура внутри не меняется: `source/`, `translation/`,
`variants/` в корне.

1. **Токен**: на github.com → Settings → Developer settings →
   **Fine-grained personal access tokens** → Generate new token. В «Repository
   access» выберите **только** репозиторий перевода, в «Permissions» дайте
   **Contents: Read and write**. Больше ничего не нужно.
2. В приложении нажмите **«Открыть с GitHub»**, впишите `владелец/репозиторий`
   (или ссылку), при желании ветку, и вставьте токен. Токен хранится только в
   вашем браузере и отправляется только на GitHub.
3. Работа выглядит так же, как с локальной папкой. Каждое нажатие «Сохранить» —
   это **коммит** в выбранную ветку (сообщение вида `Вычитка: translation/05a.md`),
   так что история правок ведётся сама.

Если файл успел измениться на GitHub после того, как вы открыли раздел (кто-то
сохранил раньше вас), запись не пройдёт молча: приложение предупредит и ничего
не затрёт — переоткройте раздел и повторите правку.

Локальная папка и GitHub-режим прекрасно сочетаются: на компьютере работаете с
локальной папкой и пушите через git, с телефона — открываете тот же репозиторий
через «Открыть с GitHub».

## Как устроена папка проекта

Минимум для работы — папки `source/` и `translation/`. Папка `variants/`
необязательна (нужна, только если хотите видеть рядом другие варианты перевода).

```
мой-перевод/
├─ vychitka.html              ← само приложение
├─ source/                    ← арабский оригинал
│  ├─ 390-393.md
│  └─ Глава 1.md
├─ translation/               ← ваш перевод (заголовок — внутри файла)
│  ├─ 390-393.md
│  └─ Глава 1.md
└─ variants/                  ← необязательно: другие варианты для сравнения
   ├─ 390-393 gemini.md
   ├─ 390-393 chatgpt.md
   └─ Глава 1 мой.md
```

### Метка, заголовок и спаривание

У каждого блока есть **метка** — свободное короткое имя (`390-393`, `Глава 1`,
`фатава 7`). Метка — это **имя файлов** блока и основа спаривания и сортировки.
Отдельно есть **заголовок** — человекочитаемое название; оно хранится **внутри**
перевода (во фронтматтере, поле `title:`), а не в имени файла.

| Папка          | Имя файла              | Что в имени            |
|----------------|------------------------|------------------------|
| `source/`      | `<метка>.md`           | метка                  |
| `translation/` | `<метка>.md`           | метка (заголовок — в `title:`) |
| `variants/`    | `<метка> <ярлык>.md`   | метка + ярлык источника |

- `source/<метка>.md` ↔ `translation/<метка>.md` спариваются по совпадению метки.
- Вариант относится к блоку, чья метка — **самый длинный префикс** имени файла
  варианта; остаток (после пробела) — ярлык. Так метки с пробелами не путаются:
  `Глава 1 gemini.md` → блок `Глава 1`, ярлык `gemini`; `Глава 10 мой.md` → блок
  `Глава 10`, не `Глава 1`.
- Список блоков **сортируется по метке по алфавиту**. Чтобы сменить порядок —
  смените метку (переименование блока, см. ниже).
- Ярлык варианта свободный: `gemini`, `chatgpt`, `claude`, `мой черновик`.

## Форматы файлов

### `source/` — оригинал

Арабский текст в `.md`. Текст бьётся на **секторы** (см. ниже). Служебные
элементы:

- **Markdown-заголовки** (`#`, `##`, `###` …) — это структурные заголовки самого
  текста, их переводят. Помечаются фоном и боковой полосой (с градацией по
  уровню) — как структурный сектор.
- **Пометка страницы** — markdown-комментарий вида `<!-- ص: 390 -->`. Бледная,
  только в оригинале; в перевод не попадает и сектором не считается.
  Необязательна — нет пометок, текст идёт сплошняком.

Оригинал **редактируемый** — у него своя кнопка «Сохранить» (пишет в
`source/<метка>.md`, без фронтматтера). Это нужно, чтобы править границы
секторов (добавлять/убирать пустые строки) и приводить число абзацев оригинала в
соответствие с переводом: один арабский абзац часто соответствует двум русским.
Сохранение — только по кнопке или `Ctrl/Cmd+S`; при несохранённых правках —
предупреждение, как у перевода.

### `translation/` — перевод

`.md` с YAML-фронтматтером сверху и телом перевода снизу. Пустая заготовка:

```markdown
---
title: Виды поклонения сердца
status: todo
glossary_checked: false
---

```

- `title` — заголовок блока (показывается в панели, правится прямо там). Меняется
  **без** переименования файлов.
- `status` — `todo` → `draft` → `review` → `verified`. Меняется кнопками в
  приложении.
- `glossary_checked` — отметка проверки по глоссарию.

При сохранении приложение меняет только тело и нужные поля, остальной
фронтматтер сохраняется как есть (старый ключ `pages` тоже сохранится, но он
больше не идентификатор — имя файла = метка).

**Маркер открытого вопроса.** Если по ходу перевода вы пишете `==?ваш вопрос==`,
он подсвечивается жёлтым. Пока в тексте есть хоть один такой маркер, статус
`verified` поставить нельзя — сначала закройте все вопросы.

### `variants/` — другие варианты (необязательно)

Обычный текст/`.md` — вариант перевода того же блока от другой модели или
источника. Показывается в отдельном окне рядом для сравнения; правки можно
сохранить обратно в этот файл. На итог не влияет — итог только в `translation/`.

## Как пользоваться

1. Слева — список блоков со значком статуса и заголовком. Выберите блок.
2. Справа — окна-слоты: оригинал (арабский, RTL) и ваш перевод (RU). Правьте перевод.
3. Кнопками в шапке меняйте статус готовности; сохраняйте кнопкой или `Ctrl/Cmd + S`.

### Сохранение и несохранённое

- У **каждого окна** в шапке — своя кнопка **«Сохранить»**: пишет только файл
  этого окна (оригинал → `source/`, перевод → `translation/`, вариант →
  `variants/`). `Ctrl/Cmd + S` сохраняет окно, в котором стоит курсор.
- **«Сохранить всё»** в шапке блока — сохраняет разом все окна с несохранёнными
  изменениями, включая черновики **других разделов** (например, пришедшие в фоне
  ответы модели).
- Несохранённое помечается **фиолетовым**: рамка окна, его кнопка «Сохранить»,
  точка у блока в списке слева. Фиолетовый — служебный цвет «несохранено», он
  не пересекается с цветами статусов.
- Точка **●** в шапке блока — индикатор «есть несохранённое»; клик по ней
  показывает список: какие именно окна и в каких разделах не сохранены.
- При уходе из вкладки или переключении блока с несохранёнными правками
  приложение предупредит; правки при переключении разделов не теряются —
  держатся в памяти до сохранения (но пропадут при закрытии вкладки).

### Поиск и замена (Ctrl+F, Ctrl+H)

`Ctrl/Cmd + F` открывает строку поиска; `Ctrl/Cmd + H` — её же, но с курсором
сразу в поле **«Заменить на»** (замена — правее поиска, за разделителем).
Поиск идёт **в одном окне** — том, где последним был курсор (надпись справа
подсказывает, в каком); кликните в другое окно — поиск перейдёт туда.

- Все совпадения подсвечиваются голубым, текущее — ярче; счётчик показывает
  «2 из 7». `Enter` — следующее, `Shift+Enter` — предыдущее (или кнопки ↑ ↓).
- Кнопка **Аа** — учитывать регистр (по умолчанию регистр не различается).
- **«Заменить»** меняет текущее совпадение, **«Заменить всё»** — все сразу.
  Замена не трогает фронтматтер и пишется только в поле — файл меняется лишь
  после обычного сохранения. Отмена работает как всегда: `Ctrl+Z` в окне
  («Заменить всё» откатывается одним нажатием).
- Если перед `Ctrl+F` в окне был выделен текст — он сразу подставится в поиск.
- `Esc` закрывает строку и возвращает курсор в окно.

### Вставка разметки (жирный, сноски…)

Кнопки **Ж К Ч Сноска ==?==** вверху вставляют разметку в то окно, где стоит
курсор, — синтаксис markdown знать не нужно:

- **Ж** — жирный `**текст**` (`Ctrl+B`), **К** — курсив `*текст*` (`Ctrl+I`),
  **Ч** — подчёркнутый `<u>текст</u>` (`Ctrl+U`; в markdown подчёркивания нет,
  это допустимая HTML-вставка). Выделите текст и нажмите кнопку — текст
  обернётся; без выделения курсор встанет между скобками.
- **==?==** — маркер открытого вопроса вокруг выделенного.
- **Сноска** — самое полезное: приложение само подберёт свободный номер,
  вставит ссылку `[^N]` в месте курсора и заготовку определения `[^N]: ` в
  конце текста. Если открыто окно «Сноски», следящее за этим окном, — курсор
  сразу перейдёт туда, остаётся набрать текст сноски.

### Новый блок, заголовок, переименование

- **`Новый раздел`** вверху — создать блок из вставки: вставьте оригинал, укажите
  **метку** (короткое имя, напр. `Глава 1`) и **заголовок**. Приложение само
  создаст `source/<метка>.md` и `translation/<метка>.md` — про папки и markdown
  знать не нужно. Метка должна быть непустой, уникальной и без символов
  `/ \ : * ? " < > |`.
- **Заголовок** блока правится прямо в шапке (поле сверху) — он пишется в `title:`
  перевода, файлы при этом **не** переименовываются. Это частая безопасная правка.
- **`переименовать`** в шапке — сменить **метку** (редко, осознанно): меняет имя
  файлов и порядок в списке. Приложение согласованно переименует оригинал, перевод
  и все варианты блока (сначала пишет новый файл, потом удаляет старый).
- **`разделить`** в шапке — разрезать слишком большой раздел на два (см. ниже).

### Деление раздела

Кнопка **`разделить`** в шапке блока режет раздел на два по **активному сектору
оригинала**:

1. Сохраните всё (при несохранённых правках деление не запустится — приложение
   скажет, что именно сохранить).
2. Поставьте курсор в абзац **оригинала**, с которого должен начаться новый
   раздел. Этот сектор и всё после него уйдут в новый раздел; всё до него
   останется в исходном.
3. Нажмите `разделить`.

Что произойдёт:

- Новый раздел получает метку `<метка>-2` (если занята — `-3` и т.д.), заголовок
  «<старый заголовок> 2» и тот же статус. Создаются `source/<метка>-2.md` и
  `translation/<метка>-2.md`.
- **Перевод режется по сектору с тем же порядковым номером**, что и в оригинале —
  поэтому перед делением стоит выровнять число абзацев слева и справа.
- **Сноски распределяются по частям**: определение `[^id]: …` уходит в ту часть,
  где на сноску ссылаются; ничейные определения остаются на своём месте.
- Варианты (`variants/`) не трогаются — они остаются у исходного раздела.
- Порядок безопасный: сначала создаётся и проверяется новый раздел, и только
  потом обрезается исходный.

### Окна (слоты)

Раскладку окон вы настраиваете под себя — она запоминается в браузере (не в
репозитории) и работает на всех блоках:

- **`＋ Окно`** вверху — добавить окно: оригинал, перевод, **вариант**
  (для варианта укажите ярлык: `gemini`, `chatgpt`, `claude`, `мой черновик`…)
  или **сноски** (окно с определениями сносок — см. раздел «Сноски»).
- В заголовке окна: **⟨** — свернуть в корешок, **◂ ▸** — передвинуть, **×** — убрать.
- Окно-вариант редактируемое; его кнопка **«Сохранить»** пишет в
  `variants/<метка> <ярлык>.md` (папка `variants/` создаётся при необходимости).
  Можно просто вставить текст из другого чата и сохранить.
- Если для блока нет файла перевода — окно перевода предложит **«Начать перевод»**
  (спросит заголовок и создаст заготовку в `translation/`).

**Сворачивание.** Окна в ряду делят ширину. Кнопка **⟨** сворачивает любое окно в
узкий вертикальный корешок на его месте — освободившуюся ширину забирают соседи.
Клик по корешку разворачивает окно обратно; содержимое не теряется.

**Режим чтения.** Кнопка **`⊟ Шапки`** вверху прячет служебные шапки всех окон —
остаётся чистый текст для спокойного чтения. Исключение: у окна с несохранёнными
правками кнопка «Сохранить» остаётся видимой, чтобы несохранённое не потерялось
из виду. Повторное нажатие (`⊞ Шапки`) возвращает шапки; режим запоминается в
браузере.

На узком экране окна выстраиваются вертикально, в столбик.

### Секторы (абзацы)

Внутри блока текст делится на **секторы** — абзацы между пустыми строками; каждый
markdown-заголовок (`#`, `##`, …) тоже отдельный сектор. Секторы во всех окнах
стоят по порядку друг напротив друга. Кликните/поставьте курсор в один сектор —
и **тот же по счёту** сектор подсветится во всех окнах сразу (оригинал, перевод,
варианты), чтобы глаз держался на одном месте.

Заголовки помечаются фоном и боковой полосой на строке (размер шрифта не меняется,
чтобы курсор и отмена в редактируемых полях работали как обычно), с градацией по
уровню — структура видна с одного взгляда в любом окне.

Граница сектора — **одна пустая строка** (двойных пустых как отдельного понятия
нет). Границы вы задаёте сами набором текста (добавили/убрали пустую строку) —
никакой служебной разметки в файле не появляется, перевод остаётся чистой прозой.
Если число абзацев слева и справа разошлось — окна показываются по порядку, а где
пары нет, сектор просто не подсвечивается.

### Сноски

Сноски — обычные markdown-сноски: в тексте ссылка `[^1]` (или `[^р1]`, id —
любой), внизу файла определения:

```markdown
Текст с сноской[^1].

[^1]: Текст самой сноски.
```

Чтобы не прыгать вниз файла, добавьте **окно «Сноски»** (`＋ Окно` → Сноски):

- Окно показывает определения `[^id]: …` того окна, где **последним был фокус**
  (оригинал, перевод или вариант) — подпись в шапке («Сноски · перевод»)
  говорит, чьи сноски показаны. Кликнули в оригинал — окно показывает сноски
  оригинала (и переключается на RTL).
- Окно **редактируемое**: правки переписывают хвост определений в ведомом окне,
  основной текст не трогается. Своего файла у окна сносок нет — его кнопка
  «Сохранить» (и `Ctrl/Cmd + S`) сохраняет **файл ведомого окна**.
- Поставьте курсор на ссылку `[^id]` в тексте — окно сносок **прокрутится к её
  определению**.

При **делении раздела** определения сносок автоматически уходят в ту часть, где
на них ссылаются (см. «Деление раздела»).

## Запрос перевода у модели (API, необязательно)

Окно-вариант умеет само запросить перевод оригинала у выбранной модели через
**OpenRouter** (один ключ, любые модели). Это просто ещё один способ заполнить
окно варианта — ответ вы смотрите и **сохраняете сами**, автосохранения нет.

1. **⚙ Настройки** вверху — введите **ключ API** OpenRouter (хранится только в
   вашем браузере, в репозиторий не попадает), список моделей и **команды
   запроса** (см. ниже).
2. В окне-варианте выберите **команду** (выпадашка рядом с «Запросить») и
   **модель** (поле; можно вписать свою строку). Слот запоминает свою команду и
   модель — можно «настроить» один вариант под тафсир, другой под перевод.
3. Нажмите **«Запросить»** — откроется окно с уже подставленным текстом (по scope
   команды); его можно разово подправить перед отправкой.
4. **«Отправить»** → окно запроса закрывается, запрос идёт **в фоне** — можно
   продолжать работать. В самом окне-варианте появляется полоса состояния:
   спиннер, название модели, **счётчик времени** и кнопка **«Отменить»** (модели
   бывают медленными — видно, что процесс идёт). Когда ответ готов, он сам
   подставляется в окно. Проверьте и нажмите **«Сохранить»**, чтобы записать в
   `variants/<метка> <ярлык>.md`.

Точные идентификаторы моделей берите на openrouter.ai/models — они со временем
меняются; в настройках и в поле слота можно вписать любую строку модели.

### Команды запроса

В Настройках — библиотека **команд**. Команда = имя + текст инструкции + **что
слать модели** (scope):

Скоп задаётся **двумя независимыми осями** — любое сочетание доступно:

- **Источник**: `Оригинал` / `Перевод` / `Оригинал + перевод` / `Этот слот`
  (содержимое того окна, **из которого** запущена команда — ответ кладётся туда же);
- **Охват**: `целиком` / `активный сектор` (абзац, где стоит курсор; если сектор
  не выбран — приложение мягко попросит выбрать, запрос не уйдёт).

Так доступны и «оригинал + перевод — активный сектор», и «этот слот — активный
сектор», без дыр. В тексте команды: `{text}` — выбранный источник в выбранном
охвате; для «оригинал + перевод» — `{original}` и `{translation}` (оба в выбранном
охвате). Команды можно добавлять, править и удалять; несколько примеров идут из
коробки («Перевести», «Тафсир аята», «Разбор грамматики», «Проверь мой перевод»,
«Сделай естественнее»). Ответ любой команды падает в то же окно — справочные
(тафсир и т.п.) можно просто не сохранять.

**Запрос есть у всех текстовых окон — оригинала, перевода и вариантов.** Кнопка
«Запросить» с выпадашкой команд работает везде. Но в окнах **оригинала и перевода**
результат подставляется в поле и **в файл не пишется сам**: сохранение только по
кнопке/`Ctrl+S` (модель не переписывает ваш оригинал и итог молча). В вариантах
такой защиты нет — там черновики. Команды над оригиналом удобны для очистки текста
после распознавания, разбивки на абзацы, удаления колонтитулов и т.п.

Запрос идёт в фоне, привязан к своему разделу и окну: можно уйти в другой раздел,
ответ придёт и **дождётся вас** там, откуда был отправлен. Несохранённое (ответы
и ваши правки) держится в памяти на время сессии и не теряется при переключении
разделов — но не сохранено в файл, пока вы не нажали кнопку/`Ctrl+S`, и пропадёт
при закрытии вкладки.

**Двухпроходный перевод без копирования.** scope «текст этого слота» позволяет
гонять прогоны цепочкой в одной колонке: оригинал → команда «творческий» (scope
«оригинал») кладёт черновик в вариант → команда «технический» (scope «текст этого
слота») дорабатывает его на месте. Кнопка **«Взять в перевод»** в варианте
переносит его текущий текст в окно перевода (в поле, без записи в файл) — удобно,
если первый прогон делали в варианте, а второй ведёте уже в переводе.

### Своды правил перевода

В Настройках можно завести **своды правил** — именованные тексты (терминология,
регистр, стиль, чего избегать). Сводов может быть несколько («Академический»,
«Для широкого читателя», «В стиле такого-то» — стиль задаётся **описанием
словами**, имя свода лишь ярлык).

У каждой команды есть выбор **свода правил** (или «без свода»). При запросе модели
уходит: текст выбранного свода + промт команды + текст по scope. Команда на свод
**ссылается**, а не копирует его: поправили свод один раз — обновилось во всех
командах, которые на него опираются. Один свод можно повесить на несколько команд.

Запросы к моделям работают при открытии **по ссылке (Netlify)**; из локального
файла (`file://`) браузер их блокирует — приложение покажет понятное сообщение, а
ручная вставка и сохранение работают всегда. Ошибки (нет ключа, лимит, нет сети)
показываются в окне запроса, приложение не падает.

## Приватность

Файлы перевода никуда не отправляются — приложение работает локально с вашей
папкой. Единственная внешняя отправка — сам запрос к OpenRouter, и только когда вы
нажали «Отправить». Ключ API хранится только в вашем браузере (`localStorage`), в
репозиторий не попадает. Никакой телеметрии.
