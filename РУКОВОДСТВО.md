# Руководство пользователя — «Вычитка»

Полная шпаргалка по всем кнопкам и режимам. Актуально на 2026-06-17.
(README — более старый обзор; здесь — текущее состояние целиком.)

«Вычитка» — инструмент для письма и сверки перевода: рядом несколько окон
(оригинал, перевод, варианты, результат…), данные — `.md`-файлы на диске. Работает
в десктопных **Chrome/Edge** (локальные файлы), плюс режим **GitHub** (где угодно,
в т.ч. телефон).

---

## 1. Как открыть текст (верхняя панель)

- **Открыть папку** — папка перевода со структурой `source/` + `translation/`
  (+ необязательно `variants/`). Выбираешь именно ВЕРХНЮЮ папку, что их содержит.
  Даёт параллельный вид: оригинал ↔ перевод, варианты. Пары — по совпадению имени
  файла (`source/маджлис.md` ↔ `translation/маджлис.md`).
- **Открыть с GitHub** — та же папка, но в репозитории (нужен токен). Работает на
  телефоне; сохранение = коммит.
- **Открыть книгу** — папка-книга читалки: `book.json` + языковые папки
  `<lang>/NN.md`. Правишь книгу на месте.
- **Открыть файл** — один `.md`, изолированно (без папок). Браузер **не даёт**
  открыть соседние файлы/папку от одного файла — для параллельного вида с
  вариантами используй «Открыть папку».
- **Черновик** — при запуске без открытого хранилища сразу даются пустые
  редактируемые окна. Папку/файл выберешь при первом «Сохранить».
- **✕ Закрыть** — сохранить всё и закрыть текущую папку/книгу/файл, вернуться к
  пустым окнам (черновику).
- Кнопка **«Открыть „…“»** при запуске — переоткрывает то, с чем работал в прошлый
  раз.

## 2. Левая панель — список блоков (☰)

Слева — список блоков (глав/диапазонов) с фильтром и значком статуса. **Кнопка ☰**
(слева вверху) скрывает/показывает эту панель. Клик по блоку — открывает его в окнах.
*(Если «ничего не открывается» — проверь, не скрыта ли панель: нажми ☰ и кликни блок.)*

## 3. Окна (слоты)

- **＋ Окно** — добавить окно. Типы: **Оригинал (AR)**, **Перевод (RU)**,
  **Вариант** (с ярлыком: gemini, chatgpt…), **Сноски**, **Результат (рендер)**,
  **Структура (план)**, **Тезисы (каркас)**, **Корбоард (карточки)**.
- Писательские окна (следят за окном письма, где курсор):
  - **Структура** — план по заголовкам: клик — прыжок, **▲▼** — переставить раздел,
    **▸/▾** — свернуть подразделы.
  - **Тезисы** — отдельный каркас аргументации (хранится по документу, в текст
    статьи не попадает).
  - **Корбоард** — карточки разделов (`##`): клик — прыжок, **перетаскивание мышью**
    — перестановка раздела (текст едет вместе с заголовком).
- В шапке окна: **⟨** свернуть в корешок, **◂ ▸** переместить, **×** убрать,
  **Сохранить** (пишет только в свой файл) и точка несохранённого.
- **⊟ Шапки** — режим чтения: прячет служебные шапки окон.
- Несохранённое помечается фиолетовым.

## 4. Письмо и разметка (панель разметки)

Действует на окно, где курсор. Группы:

- **Формат:** **Ж** жирный (`**`), **К** курсив (`*`), **Ч** подчёркнутый (`<u>`).
- **Блок:** **Заголовок** (`## `), **❝ Цитата** (`> `), **• Список** (`- `),
  **1. Список** (`1. `), **Callout ▾** — выноска в стиле Obsidian (`> [!тип]`,
  12 типов: цитата, примечание, важное, пример…).
- **Вставка:** **Сноска** (`[^N]` + заготовка определения внизу), **стр.** (маркер
  страницы, только в оригинале), **==?==** (открытый вопрос — жёлтым, блокирует
  статус verified), **Заметка** (личная, см. §8).
- **Связи:** **🔗 Связать** и **⌗ Якоря** (см. §7) — появляются в нужных режимах.

Горячие клавиши: **Ctrl+B/I/U** — жирный/курсив/подч.; **Ctrl+S** — сохранить окно
с курсором; **Ctrl+F** — поиск, **Ctrl+H** — замена.

## 5. Показ результата (как будет выглядеть)

Кнопка показа результата (верхняя панель) циклит 3 режима:
- **Разметка** — обычное поле (видишь символы разметки);
- **Сплит** — сверху правка, снизу **живой рендер** (обновляется по мере набора);
- **Рендер** — только результат во весь слот.

Плюс отдельное **окно «Результат»** (＋ Окно → Результат): сбоку, во всю высоту,
показывает живой рендер того окна, где курсор. Удобно: пишешь в одном окне,
результат рядом.

## 6. Сноски

Кнопка **Сноска** ставит `[^N]` в тексте и кладёт заготовку `[^N]: …` в конец.
Окно **Сноски** показывает определения активного окна, редактируемо; клик по `[^N]`
прокручивает к определению.

## 7. Сектора и пары (сверка)

**Сектор** — абзац (кусок между пустыми строками). Сектора оригинала и перевода
**парятся**, чтобы видеть соответствие.

- **По порядку** (по умолчанию): 1-й к 1-му, 2-й ко 2-му… Если объёмы совпадают.
- **По якорю** `<!-- sNNN -->`: если над абзацем стоит якорь, пара идёт **по
  совпадению номера** в разных окнах, а не по порядку. Нужно, когда абзацев разное
  число (своя проза + вставные цитаты).
- **🔗 Связать** (видна при 2+ окнах): ставишь курсор в абзац → жмёшь → ему даётся
  свободный якорь, такой же кладётся в другое окно — туда впишешь пару.
- **⌗ Якоря** (в режиме книги): проставить якоря всем абзацам без якоря разом.

Подсветка активного сектора (фон) показывается только при 2+ окнах — в одном окне
письма её нет.

## 8. Заметки (личные)

**Заметка** оборачивает текст в `<!-- note --> … <!-- /note -->`: видна тебе
(бирюзовое поле), но **не считается сектором** (парность не сдвигает) и **в читалке
при чтении скрыта**. Для рабочих пометок «на полях».

## 9. Статусы перевода

В шапке блока: **todo → draft → review → verified**. Клик меняет статус и сразу
сохраняет. **verified нельзя поставить**, пока в переводе есть живой `==?…==`.

## 10. Поиск и замена

**Ctrl+F** — поиск, **Ctrl+H** — замена. Работают в окне, где последним был курсор;
совпадения подсвечиваются. «Аа» — учитывать регистр; ↑↓ — переходы; «Заменить» /
«Заменить всё».

## 11. Сохранение

- У каждого окна своя **«Сохранить»**; **«Сохранить всё»** — все окна и черновики;
  **Ctrl+S** — окно с курсором.
- Несохранённое — фиолетовым; при уходе/закрытии предупреждает.
- Итоговый файл — `translation/` (в папочном режиме). Оригинал и варианты тоже
  сохраняются в свои файлы.

## 12. Запрос перевода у модели (API, необязательно)

В окне (оригинал/перевод/вариант) — выбор **команды** и **модели** + кнопка
**Запросить**: модель переводит, ответ кладётся в окно (обычно вариант). Ключ
**OpenRouter** — в **⚙ Настройки**, хранится только в браузере. Там же библиотека
команд и своды правил перевода.

## 13. Библиотека читалки

- **Открыть книгу** — править книгу читалки на месте (см. §1).
- **Опубликовать в библиотеку** (в папочном режиме) — превратить репозиторий
  `source/translation` в книгу читалки (сектора `sNNN`, сноски, страницы, `book.json`,
  реестр). Выбираешь папку `books/` читалки, id и языки.

## 14. Где хранится и приватность

Файлы — у тебя на диске (или в твоём GitHub-репозитории). Настройки (ключ API,
раскладка окон, последняя папка) — в `localStorage`/IndexedDB браузера. Никакой
телеметрии; наружу уходит только сам вызов модели (если им пользуешься).
