# Changelog — «Вычитка»

Хронология изменений для самопогружения в контекст. Подробности механики — в
`CLAUDE.md`. Новое сверху.

## 2026-06-19

### Публикация в библиотеку: форма (раздел, обложка, аннотация, автор, теги)
- Кнопка «Опубликовать в библиотеку» вместо череды `prompt()` открывает **форму**:
  id, название, языки, **раздел** (datalist существующих категорий + ввод новой,
  вложенность через « / »), **автор**, **аннотация**, **теги**, **обложка**
  (файл-пикер, можно пропустить — читалка нарисует заглушку).
- Подсказка в форме показывает выбранную папку и **сколько книг уже в реестре**
  (страховка: 0 = выбрана не та папка `books/`). При вводе id существующей книги
  поля метаданных **подтягиваются** из её `index.json`/`book.json`.
- При повторной публикации метаданные **не теряются**: `book.json` сливается с
  прежним (наследуем `author`/`description`/чужие поля, перезаписываем только
  главы/языки/заголовок), запись `index.json` — тоже merge (категория/обложка/
  теги/автор сохраняются). Обложка пишется как `cover.<ext>` в папку книги, путь —
  в запись реестра. `publishToLibrary`/`runPublish`/`pubReadJSON`/`pubPrefillById`.
- Деплой не изменился: публикация пишет в локальную папку `books/` клона читалки;
  дальше — `git push` репозитория читалки → Netlify сам выкладывает.

### Деление раздела по курсору в любом окне (не только оригинал)
- Раньше «разделить» требовало арабский оригинал и резало по его сектору. Теперь
  `splitBlock` режет **каждое окно независимо** по активному сектору (курсор может
  стоять в переводе). Окно без такого сектора целиком остаётся в исходном разделе
  (`splitTextAtSector` возвращает пустую вторую часть). Нужно для статей, где основа
  на русском, а арабский — лишь отдельные вставные цитаты: новый раздел получит
  только русский хвост, оригинал не обязателен. Создаём файлы нового раздела только
  из непустых частей; исходный обрезаем лишь там, откуда ушёл хвост.

### Перестановка разделов ▲▼ (папочный режим)
- У раздела с переводом — кнопки **▲/▼** в списке слева (видны при наведении).
  Порядок хранится в ключе `order:` фронтматтера перевода. Сортировка списка —
  по `order` (у кого есть), иначе по метке (алфавит); дефолт = алфавитный индекс,
  поэтому переставленный блок чисто встаёт между соседями. Перестановка пишет
  `order` **только в один файл** (середина между соседями) — важно для GitHub
  (каждая запись = коммит). `moveBlock`/`sortBlocks`/`blockOrder`. Источник-only
  раздел (без перевода) переставить нельзя — порядок негде хранить. Под фильтром
  кнопки скрыты (соседи в списке ≠ соседи в полном порядке).

## 2026-06-18

### Сворачиваемые callout’ы (как в Obsidian)
- В рендере (показ результата / окно «Результат») callout’ы с `+`/`-` после типа —
  сворачиваемые: `> [!тип]+` раскрыт по умолчанию, `> [!тип]-` свёрнут (клик по
  заголовку разворачивает). Без знака — как раньше, статичный. Реализовано на
  `<details>`/`<summary>` (`pvBlocks`, регэксп `[!тип]([+-]?)`). Тело свёрнутого
  скрыто явным CSS (не полагаемся на UA-стиль `<details>`). **Тот же словарь и в
  читалке** (`parser.js`, `style.css`) — книга рендерит точно так же.

### Писательские функции (E): окно «Корбоард»
- Новое окно через ＋Окно → **«Корбоард»**: карточки разделов (`##`/`###`) ведомого
  окна письма — заголовок + начало текста. Следит за окном письма (как
  структура/результат/сноски — `refreshCorkboard`/`fnFollowSlot`).
- **Клик по карточке** — прыжок к разделу в окне письма.
- **Перетаскивание мышью** (HTML5 drag-and-drop) — перестановка раздела: карточка
  бросается на другую → раздел (от заголовка до следующего) переезжает целиком,
  текст едет вместе с заголовком (`reorderSections`). Целостность проверена:
  тот же мультимножество символов, без потерь/дублей. Раздел — плоский (как в
  «Структуре»): подразделы переставляются независимо.

## 2026-06-17

### Тумблер синхронизации на окне (⇅)
- У окон источник/перевод/вариант — кнопка **⇅**: выключить синхронную прокрутку/
  подсветку для этого окна. Выкл — окно не прокручивается к парному сектору и не
  двигает другие (независимо); удобно при вычитке статей, где синхронизация местами
  мешает. Состояние в раскладке (`spec.nosync`). Затрагивает `syncActiveFromEditor`/
  `scrollTargetsFrom`/подсветку sec-active.

### Топбар компактнее + меню «Открыть ▾»
- Кнопки переносятся на 2-ю строку (flex-wrap) — крайние больше не пропадают за
  краем; компактнее (меньше padding/шрифт). 4 кнопки «Открыть…» свёрнуты в одно
  меню **«Открыть ▾»** (Папка/GitHub/Книга/Файл).

### Писательские функции (C): окно «Тезисы / план»
- Новое окно через ＋Окно → **«Тезисы (каркас)»**: редактируемый каркас
  аргументации (список тезисов, чек-лист `- [ ]` / `- [x]`). Хранится **отдельно
  по документу** (localStorage, ключ из режима+источника+метки), в текст статьи не
  попадает; подтягивается при смене документа и после перезагрузки. Роль `theses`,
  `thesesKey`/`loadThesesText`/`saveThesesText`. *MVP: привязка тезиса к секции и
  авто-«раскрыто» — следующий заход.*

### Писательские функции (B): свёртка секций в «Структуре»
- В окне «Структура» у `##`-раздела с подразделами — кнопка **▸/▾**: свернуть/
  развернуть его `###`-дочерние (виден скелет нужного уровня). Состояние —
  `state.outlineCollapsed`. *Примечание: настоящего инлайн-сворачивания в самом
  `<textarea>` нет (нужен код-редактор; манипуляции текстом рискованны) — скелет
  живёт в панели «Структура».*

### Писательские функции (A): окно «Структура» (план)
- Новое окно через ＋Окно → **«Структура (план)»**: узкая панель сбоку с планом
  по заголовкам (`##`/`###`) окна письма, следит за ним (как окно-результат/сносок).
- **Клик по пункту** — прыжок к разделу в окне письма (курсор + прокрутка).
- **▲▼** — переставить раздел (раздел = от заголовка до следующего заголовка; текст
  едет вместе с заголовком). Целостность проверена: без потерь/дублей.
- Роль `structure`, `refreshOutline`/`jumpEditorTo`/`moveSection`/`headingSections`.
  Это «компоновка частей» из писательского набора (дальше: B свёртка, C тезисы, E корбоард).

### Монолингвальные книги (один язык)
- Книга читалки с одним языком (`languages:["ru"]`) открывается **в одном
  главном окне** (перевод), пустое окно оригинала **прячется** (`book.single`,
  `hideSource` в renderSlots). Двуязычные — без изменений. RTL — из `book.rtl`.

### Открыть один файл (режим file)
- Кнопка **«Открыть файл»**: выбрать один `.md`/`.txt` (`showOpenFilePicker`) и
  править его напрямую, без папочной структуры; сохранение пишет в тот же файл.
  `state.mode="file"`, мини-адаптер `state.fs` читает/пишет прямо в handle; текст
  сырой (без фронтматтера). Переоткрытие — как у папки/книги (IDB `file`).

### Связывание пар, режим «Результат», перекомпоновка панели
- Кнопка **«🔗 Связать»** (видна при 2+ окнах): текущему абзацу присваивается
  свободный якорь `sNNN`, такой же кладётся в конец других окон — туда вписать
  пару. Так нужный абзац соединяется с парой по ID, а не по порядку (`linkSector`).
- Кнопка показа результата (topbar) циклит 3 режима: **Разметка → Сплит
  (правка сверху + живой рендер снизу) → Рендер (только результат)**. Рендер
  read-only: жирный/курсив/подч./заголовки/цитаты/списки/callout’ы; якоря и
  служебные комментарии скрыты, заметки — приглушённым блоком
  (`renderPreviewHtml`/`cyclePreview`/`refreshPreview`). Полноценный WYSIWYG-ввод
  (правка прямо по рендеру, contenteditable) — отдельная задача.
- **Окно-результат** (роль `result`, добавляется через ＋Окно): отдельное окно
  сбоку, во всю высоту, показывает живой рендер последнего активного окна письма
  (следует за ним, как окно сносок — `refreshResults`/`fnFollowSlot`). Read-only,
  без сохранения. Так пишешь в одном окне, результат — рядом, место не урезается.
- Панель разметки перекомпонована: группы Формат / Блок / Вставка / Связи
  (Связать, ⌗Якоря).
- Фиксы: подсветка активного сектора — только при 2+ окнах (в одном окне не
  заливает текст); `restoreSlot` в черновике/книге не показывает «Перевод ещё не
  начат» вместо пустого поля при удалении окна.

## 2026-06-16

### Опубликовать в библиотеку (folder → книга читалки)
- Кнопка **«Опубликовать в библиотеку»** (папочный режим): конвертирует
  `source/translation`-репозиторий в формат читалки. Порт логики `import-tam.js`
  внутрь вычитки (`pubParse`/`pubConvert`): сектора → `<!-- sNNN -->`, заголовки →
  `s00Na/b` жирным, `[^id]`→`[^N]`+`<!-- fnN -->`, `<!-- ص: N -->`→`<!-- pN -->`,
  раз-линковка `[[вики]]`, выброс арабских строк из ru-стороны. Оригинал пишется
  только при посекторном паритете (иначе глава ru-only). Пишет
  `books/<id>/<lang>/NN.md`, `book.json`, обновляет реестр `index.json`.

### Режим «книга» (формат читалки) — правка библиотеки на месте
- Кнопка **«Открыть книгу»**: открыть папку-книгу читалки (`book.json` + языковые
  папки `<lang>/NN.md`) и править на месте, без конвертации. `state.mode="library"`,
  `state.book`.
- Роли сопоставлены языкам: `languages[0]`→`source`, `languages[1]`→`translation`,
  `variants/`→варианты. Блоки = главы из `book.json.chapters` (порядок сохраняем).
- `title`/`status` пишутся в `book.json` (без фронтматтера); язык-файлы — сырой
  текст. RTL — из `book.rtl` (арабский не захардкожен).
- Папочный режим `source/translation` и черновик не затронуты (ветвление по
  `state.mode`). Адаптер `state.fs` переиспользован.

### Явные сектора sNNN (парность по ID)
- `parseBlocks` распознаёт якоря `<!-- sNNN -->` / `<!-- sNNNa -->`; сектор получает
  `id`. Новый `secKey(b) = id ?? idx`: анкорные секторы парятся между языками по ID,
  старые (без якорей) — по порядку (обратная совместимость).
- На `secKey` переведены `sectorRange`/`sectorAtCaret`/подсветка в `updateBackdrop`/
  `scrollSlotToSector`. «Разделить раздел» выключено в режиме книги.
- Кнопка **«⌗ Якоря»** (только в книге): проставляет `<!-- sNNN -->` абзацам без
  якоря, продолжая нумерацию от максимума; существующие ID не трогает.
- Зачем: гибридные книги (моноязычный текст + вставные параллельные цитаты) и
  публикация недопереведённых книг.

### Markdown-редактор (панель разметки)
- Сгруппированная панель: текст (Ж/К/Ч), блоки (Заголовок/❝Цитата/Списки/Callout▾),
  вставка (Сноска/стр./==?==/Заметка).
- Цитата `> `, заголовок `## `, списки `- `/`1. `, callout’ы Obsidian
  `> [!тип]` (пикер 12 типов; тип парсится обобщённо). Подчёркивание — `<u>`.

### Заметки (непарные секторы)
- `<!-- note --> … <!-- /note -->`: текст виден, но абзацы внутри не считаются
  секторами (парность не сдвигается). Единый блок, бледная заливка. Личные
  редакторские примечания — читалка их при чтении прячет.

### Черновик без папки + «Закрыть папку»
- При запуске сразу пустые редактируемые окна; папку/книгу выбираем при первом
  «Сохранить» (создаётся блок, переход в папочный режим).
- «Закрыть папку» — автосохранение всего и возврат к черновику.

### Фиксы
- Раздел/окна открываются **с начала во ВСЕХ окнах** (Chrome ставил курсор в конец
  после set value → прокрутка вниз). Раньше чинилось только для перевода.

## Связанный проект — читалка (`chitalka-parallel`, ветка master)
Вычитка = редактор её библиотеки. Парсер читалки рендерит тот же словарь
форматирования (`<u>`, `> цитата`, списки, callout’ы), прячет `<!-- note -->`,
и недопереведённые секторы показывает оригиналом во всю ширину с тонкой меткой
«идёт перевод». Контракт формата — в `CLAUDE.md` читалки.
